# 전체 구조 요약

======================================================================

            [Flutter App] ← 사용자: 고령자, 봉사자
           ↓↑             ↓
      Firebase (Auth / Firestore / FCM)   ↔   [FastAPI Server on GCP]
                                 ↙︎           ↘︎
                  Google Speech-to-Text      Gemini API

======================================================================

사용자 정보/기본 기능은 Flutter ↔ Firebase
음성 처리 / 자연어 해석 / 포인트 계산 등 무거운 로직은 Flutter → FastAPI → Firebase

# 📌 추가 구현 요소: 지자체 연동 기능

## ✅ 목표

고령자와 봉사자의 요청, 매칭, 후기 등의 활동 이력을  
지자체 담당자가 열람하고, 정책 수립이나 복지 사업에 활용할 수 있도록  
**지자체 전용 관리자 기능을 제공**하는 것이 목표입니다.

---

## 🧩 주요 기능

### 1. 관리자 전용 계정 체계 도입
- Firebase Auth의 사용자 `role` 필드에 `admin` 권한 추가
- 로그인 후 권한에 따라 화면/기능 분기 (고령자, 봉사자, 관리자)

### 2. 관리자 전용 대시보드 (웹 기반)
- 전체 요청 기록 열람 (시간, 위치, 참여자)
- 봉사자 활동 이력 열람 및 리뷰 확인
- 요청 응답률, 평균 이동 거리, 누적 포인트 등 통계 시각화
- 봉사자 인증/승인/차단 기능
- 후기 신고 처리 기능

### 3. 데이터 구조 확장 예시 (Firestore)

```plaintext
/users/{userId}
  → role: 'elderly' | 'volunteer' | 'admin'

/requests/{requestId}
  → requesterId, responderId, status, timestamp

/reviews/{reviewId}
  → reviewerId, targetId, score, comment

/admin_logs/{logId}
  → action: 'approve', 'block', 'view', etc.
